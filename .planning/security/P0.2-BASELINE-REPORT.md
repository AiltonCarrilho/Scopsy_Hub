- tar vulnerability chain (hardlink traversal, symlink poisoning)
- Fix requires: bcrypt@6.0.0 (breaking change from 5.x)
- Impact: Dev dependency (testing only, not production)
- **Decision:** Document as technical debt, upgrade in P1.2 sprint when bcrypt 6.x compatibility reviewed

**Rationale:**
```
Risk Assessment:
- bcrypt used in password hashing (high sensitivity)
- Breaking change requires full regression testing
- Current bcrypt 5.x is secure for production use
- tar impact: local machine only (dev/build phase, not runtime)
```
