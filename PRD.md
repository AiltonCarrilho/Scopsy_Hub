# Context

- Change request: Add an **AI Peer Review** step to the pipeline. Cases generated by `pipeline-generate.js` must be validated by "AI Reviewer Agents" (Clinical & Scientific) before reaching the human review folder.
- Expected outcome: Only cases with solid scientific grounding and correct structure reach the user.
- Constraints: Local Node.js execution.
- Assumptions: Reviewer Agents will use GPT-4o with specialized prompts.

# Impacted Areas

## Directory Structure (Updated)

```
content-pipeline/
  ├── 1-inputs/         # Input Markdown
  ├── 2-drafts/         # [NEW] Generated JSONs land here first (waiting AI review)
  ├── 2-drafts/rejected/ # [NEW] Cases rejected by AI
  ├── 3-review/         # [MODIFIED] Only AI-approved cases land here for Human Review
  ├── 4-approved/       # Ready for DB
  └── 5-archived/       # Backups
```

## New Scripts

- `scripts/pipeline-review.js`:
  - Reads from `2-drafts/`.
  - Calls AI Reviewer (Persona: "Scientific Supervisor").
  - Checks for: Theoretical accuracy, Citations (Simulated/Checked), Clinical safety.
  - If Score > 8/10: Move to `3-review/` and append `ai_review_feedback` to JSON.
  - If Score < 8: Move to `2-drafts/rejected/` with `ai_rejection_reason`.

## Modified Scripts

- `scripts/pipeline-generate.js`: Output target changes from `3-review` to `2-drafts`.

# Rejection Workflow

- If rejected, the JSON is moved to `rejected` folder.
- The JSON file will contain a new field `rejection_log` with specific reasons (e.g., "Lack of conceptual clarity", "Invalid intervention").
- User can edit the JSON (or Input) and move back to `inputs` or `2-drafts` to try again.

# Acceptance Criteria

- Must:
  - Generate script saves to `2-drafts`.
  - Review script analyzes `2-drafts`.
  - Review prompt enforces "Scientific Basis" (e.g. Beck, Young, Clark).
  - Only approved files appear in `3-review`.
